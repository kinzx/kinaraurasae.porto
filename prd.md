# PRD Portfolio Website

## Tujuan

Website ini adalah portfolio pribadi. Fungsinya untuk menampilkan profil, skill, proyek, pengalaman, dan kontak secara jelas tanpa fitur yang tidak perlu.

Dokumen ini dipakai sebagai konteks awal untuk AI ketika membuka chat baru. AI harus mengikuti isi file ini dan kondisi repo saat ini, bukan mengarang arah produk baru.

## Kondisi Repo Saat Ini

- Project memakai Next.js 16.2.10, React 19.2.4, TypeScript, Tailwind CSS 4, ESLint, Prettier, Vitest, dan Playwright.
- Struktur utama ada di `app/page.tsx`, `app/layout.tsx`, dan `app/globals.css`.
- Halaman saat ini masih template bawaan Create Next App.
- Metadata saat ini masih default: `Create Next App` dan `Generated by create next app`.
- Belum ada konten final tentang nama, bio, skill, proyek, pengalaman, foto, link sosial, atau kontak.

## Prinsip Produk

- Portfolio harus terasa personal, profesional, cepat dibaca, dan mudah dihubungi.
- Fokus utama adalah isi, bukan animasi berat atau dekorasi berlebihan.
- Jangan menambahkan database, login, dashboard admin, payment, CMS, atau API eksternal sebelum ada kebutuhan nyata.
- Jangan mengarang data pribadi. Jika data belum ada di repo atau belum diberikan user, gunakan placeholder yang jelas atau tanyakan dulu.
- Gunakan komponen dan style sesedikit mungkin selama hasilnya rapi dan responsif.

## Target Pengunjung

- Recruiter atau HR yang ingin melihat kemampuan secara cepat.
- Calon klien yang ingin tahu jasa, kualitas kerja, dan cara menghubungi.
- Rekan developer yang ingin melihat proyek dan stack yang digunakan.

## Scope MVP

Halaman utama cukup berisi:

1. Hero
   - Nama pemilik portfolio.
   - Role atau fokus kerja.
   - Kalimat singkat tentang nilai yang ditawarkan.
   - Tombol kontak dan tombol lihat proyek.

2. About
   - Ringkasan diri singkat.
   - Fokus kerja atau minat utama.

3. Skills
   - Daftar teknologi atau kemampuan.
   - Kelompokkan secara sederhana jika datanya banyak.

4. Projects
   - Nama proyek.
   - Deskripsi singkat.
   - Stack yang dipakai.
   - Link demo atau repository jika ada.

5. Contact
   - Email, WhatsApp, LinkedIn, GitHub, atau channel lain yang diberikan user.
   - Jangan tampilkan kontak palsu.

## Non-Scope Untuk Saat Ini

- Blog.
- Admin panel.
- Sistem login.
- Database.
- Payment gateway.
- Multi bahasa.
- CMS.
- Animasi kompleks.
- Tracking analytics.

Tambahkan hanya kalau user meminta dengan jelas.

## Konten Yang Masih Dibutuhkan Dari User

- Nama tampilan final.
- Role atau headline.
- Bio singkat.
- Daftar skill.
- Daftar proyek portfolio.
- Link GitHub, LinkedIn, email, WhatsApp, atau sosial lain.
- Preferensi visual jika ada.
- Foto atau avatar jika ingin dipakai.

## Arahan Desain

- Tampilan modern, bersih, dan mudah discan.
- Mobile-first dan responsif.
- Hindari layout marketing yang terlalu ramai.
- Gunakan kontras teks yang aman dibaca.
- Jangan membuat elemen dekoratif yang mengganggu isi.
- Jangan memakai warna dominan satu nada saja jika membuat desain baru.

## Arahan Teknis

- Ikuti pola App Router yang sudah ada di folder `app`.
- Jika mengubah kode Next.js, baca panduan terkait di `node_modules/next/dist/docs/` sesuai instruksi `AGENTS.md`.
- Pakai dependency yang sudah ada. Jangan tambah package baru kecuali benar-benar perlu.
- Jaga test yang sudah ada tetap jalan.
- Setelah perubahan UI, minimal jalankan:
  - `npm run typecheck`
  - `npm run lint`
  - `npm run test`

## Aturan Untuk AI

- Mulai dari membaca `prd.md`, `AGENTS.md`, `package.json`, `app/page.tsx`, `app/layout.tsx`, dan `app/globals.css`.
- Jangan keluar konteks menjadi toko online, dashboard, SaaS, atau aplikasi lain.
- Jangan membuat fitur yang tidak diminta.
- Jangan mengisi data pribadi dengan asumsi.
- Jika user meminta "buat portfolio", ubah template bawaan Next.js menjadi halaman portfolio sesuai scope MVP di atas.
- Jika data user belum lengkap, buat struktur yang rapi dengan placeholder minimal dan beri tahu data apa yang masih kosong.
